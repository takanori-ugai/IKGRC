Knowledge Graph

The knowledge graph is available in Resource Description Framework (RDF) format and published on GitHub at KGRC-RDF repository. It is provided in both TTL and N-TRIPLE formats for easy integration and handling in machine learning tasks. Additionally, it is also available in the ActionGenome format, allowing for enhanced versatility across various applications.

The knowledge graph provided for the reasoning challenge is expressed with "events" and corresponding "actions," "main objects," "target objects," "agents," "relationships between scenes (nextEvent)," "times," etc. Please see the slides below for more details.

SPARQL Endpoint

We provide an endpoint for searching the knowledge graph using SPARQL queries. Please avoid making a large number of requests that could overload the server.

Endpoint: https://kgrc4si.home.kg:7200/repositories/KGRC4SIv05

Parameters: query = {URL-encoded SPARQL query}, format = {desired data format (e.g., json, xml, csv)}

in Python Porgram with rdflib in Google Colaboratory

Here is an example of using the above SPARQL endpoint with a program (Python) in Google Colab. The same applies to your own Jupyter notebook.

  • Execution Example Using Python (rdflib)

Retrieve events during the activity of "browsing the internet"


PREFIX ex: 
PREFIX vh2kg: 
SELECT DISTINCT * WHERE {
  ex:browse_internet_scene1 vh2kg:hasEvent ?event .
  ?event vh2kg:action ?action .
}

Retrieve a list of activities


PREFIX ex: 
PREFIX vh2kg: 
SELECT DISTINCT * WHERE {
  ?activity vh2kg:virtualHome ex:scene1 .
}
